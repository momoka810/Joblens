# Webアプリケーションの公開方法

## 公開方法の選択肢

このプロジェクトは静的サイト（HTML/CSS/JavaScript）なので、以下の方法で公開できます：

1. **GitHub Pages**（無料、簡単）
2. **Netlify**（無料、簡単、自動デプロイ）
3. **Vercel**（無料、簡単、自動デプロイ）

## ⚠️ 重要な注意事項

### APIキーのセキュリティ

- **フロントエンドにAPIキーを含めることはセキュリティリスクがあります**
- 公開する場合は、APIキーが誰でも見える状態になります
- 本番環境では、サーバーサイドでAPIキーを管理することを強く推奨します

### 公開前の確認

- [ ] `api_key.txt`がGitHubにコミットされていない
- [ ] 公開後もAPIキーが含まれていないことを確認
- [ ] DifyのAPIキーに使用制限を設定することを検討

## 方法1: GitHub Pages（推奨）

### 手順

#### 1. GitHubリポジトリの設定

1. GitHubリポジトリ（`https://github.com/momoka810/Joblens`）にアクセス
2. 「Settings」タブをクリック
3. 左側のメニューから「Pages」を選択
4. 「Source」で「Deploy from a branch」を選択
5. 「Branch」で`main`を選択
6. 「Folder」で`/project`を選択（または`/ (root)`を選択）
7. 「Save」をクリック

#### 2. 公開URLの確認

数分後、以下のようなURLでアクセスできます：
```
https://momoka810.github.io/Joblens/
```

### 注意点

- `api_key.txt`はGitHubにコミットしないでください
- 公開後、ブラウザの開発者ツールでAPIキーが見える状態になります
- GitHub Pagesは静的サイトのみ対応（Pythonサーバーは不要）

## 方法2: Netlify（簡単、自動デプロイ）

### 手順

#### 1. Netlifyにサインアップ

1. https://www.netlify.com/ にアクセス
2. 「Sign up」でGitHubアカウントでログイン

#### 2. サイトをデプロイ

1. 「Add new site」→「Import an existing project」をクリック
2. GitHubを選択して、`Joblens`リポジトリを選択
3. 設定：
   - **Base directory**: `project`
   - **Build command**: （空欄のまま）
   - **Publish directory**: `project`
4. 「Deploy site」をクリック

#### 3. 公開URLの確認

数分後、以下のようなURLでアクセスできます：
```
https://random-name-123.netlify.app
```

### メリット

- 自動デプロイ（GitHubにプッシュすると自動更新）
- カスタムドメイン設定可能
- HTTPS対応

## 方法3: Vercel（簡単、自動デプロイ）

### 手順

#### 1. Vercelにサインアップ

1. https://vercel.com/ にアクセス
2. 「Sign up」でGitHubアカウントでログイン

#### 2. プロジェクトをインポート

1. 「Add New Project」をクリック
2. `Joblens`リポジトリを選択
3. 設定：
   - **Root Directory**: `project`
   - **Framework Preset**: Other
4. 「Deploy」をクリック

#### 3. 公開URLの確認

数分後、以下のようなURLでアクセスできます：
```
https://joblens.vercel.app
```

## 公開後の確認事項

### 1. 動作確認

- [ ] ページが正しく表示される
- [ ] CSSが読み込まれている
- [ ] JavaScriptが動作する
- [ ] Dify APIが呼び出される（開発者ツールで確認）

### 2. セキュリティ確認

- [ ] `api_key.txt`が公開されていない
- [ ] ソースコードにAPIキーがハードコードされていない
- [ ] 開発者ツールでAPIキーが見えることを理解している

### 3. APIキーの保護

公開後、APIキーは誰でも見える状態になります。以下の対策を検討：

- DifyのAPIキーに使用制限を設定
- 必要に応じてAPIキーを定期的に変更
- 本番環境ではサーバーサイドでAPIキーを管理

## トラブルシューティング

### CORSエラーが出る場合

Dify APIがCORSを許可していない場合、エラーが発生する可能性があります。
その場合は、サーバーサイドでプロキシを作成する必要があります。

### 404エラーが出る場合

- ファイルパスを確認（`/api_key.txt`など）
- 公開ディレクトリの設定を確認

## 推奨方法

**初心者にはGitHub Pagesが最も簡単です。**

1. 設定が簡単
2. 無料
3. GitHubと連携
4. HTTPS対応

ただし、APIキーのセキュリティリスクを理解した上で公開してください。
